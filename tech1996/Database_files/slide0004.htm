OLTP - Concurrency
lConcurrency:
–Two updates being done to the same data at the same time; if not managed carefully, one of the updates will be lost
lTwo basic strategies:
–1) Locking: Once update is in-progess, block any subsequent updates by locking database
–2) Timestamps: Check to make sure the current copy is being updated and notify if out-of-date data is being used (Usually better than locking)
