Protocols and Services
  • Protocols
    • Defines how to name machines/resources
    • Defines format for exchanging information
    • Internet = TCP/IP = Transmission Control Protocol / Internet Protocol = Low-level network protocol
  • Services
    • Protocol for how clients and servers will exchange information over TCP/IP
    • Several well-known services
      • FTP - File Transfer
      • HTTP - World Wide Web - Hyper Text/Multi-media
      • SNMP - E-Mail - Mail Exchange

Packets and Routing
  • TCP/IP “packetizes” traffic:
    • Sender sends one big piece of data
    • Network may break into smaller pieces and send each piece individually.
    • Separate packets may take different route to Receiver
    • Receiver is guaranteed to re-assemble all packets in order. (Network takes care of re-sending lost packets)
  • TCP/IP specifies packet format, naming convention for participants, routing algorithms.

Internets and Intranets
  • Both use “Internet” (TCP/IP)
  • “The Internet” is the very large, quasi-public network using the Internet protocol.
  • Intranets are private Internets - computers talk among themselves using Internet technology; but they’re not connected to the world at large
  • Most Intranets have some limited Internet access (via firewalls).

Firewalls
  • Act as traffic cops: look at sender/receiver addresses and port numbers and decide whether to route packet or not.
  • Firewalls in between network segments where they connect together.
  • Usually used to connect Intranets to Internet
