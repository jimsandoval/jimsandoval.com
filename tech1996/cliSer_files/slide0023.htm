Glossary (2)
•Usable: User-friendly. The system makes users do tasks in a way that matches the way users want to do them. A result of good task-analysis and good GUI design. Surprisingly difficult to achieve.
•Maintainable: Ability of programmers to change system. Depends on good programming style, good documentation, clear simple algorithms, consistent standards. Modular.
•Supportable: Easy to operate day-to-day. Well-defined SysAdmin roles and procedures. Centralized error handling.
•Business Rules: Processing logic and data definition exists separate from the system code in a way accessible and meaningful to non-technical team members, but still in a meaningful form for the system to use directly.
•Interaction Models: How users/systems exchange information. Four-basic types: Connectionless, conversational, request - response, publish and subscribe.
