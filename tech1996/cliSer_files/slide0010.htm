Glossary (1)
•Highly Cohesive: Individual pieces function together well as an integrated whole. Individual tasks are not sacrificed for the good of the whole.
•Loosely Coupled: Changes in implementation of one piece don’t effect other pieces.(Operating system upgrades, design changes for different modules, etc.)
•Modular: Broken down into component pieces. Easy to delegate pieces to teams. Easy to dedicate machine resources to true cause without affecting un-related processing. see Scaleable and Distributed.
•Distributed: Processing is shared across multiple machines; goal is to isolate resource use so that hardware/software can be deployed to fit needs. Modularity is logical, distribution is physical.
•Extensible: Un-anticipated requirements and/or features can be accommodated without fundamental re-design.
•Scaleable: Huge increases in users/volumes can be accommodated without fundamental re-design. Modules can be re-distributed without negative impacts.
